Human Genome Features web app

This is the final project of the advanced programming course of the Genomics bachelor degree - UniBo.

The project consists in a web application that is able to manipulate a GFF3 Dataset ( Homo_sapiens.GRCh38.85.gff3.gz) that contains all of the features of the human genome. We followed the instructions provided by the Professor, you can find them here.

A GFF3 file is a tabular dataset, organised in 9 columns:

  1. Seqid: chromosome number
  2. Source: algorithm or operating procedure that generated the feature
  3. Type: type of the DNA sequence
  4. Start: starting point of the sequence (in bp)
  5. End: end point of the sequence (in bp)
  6. Score: quality score of the feature
  7. Strand: DNA strand on which the feature is placed
  8. Phase: reading frame of the feature
  9. Attribute: additional information about the feature
The dataset that we used was published in 2013 and it has 2601849 rows. If you want to know more about GFF3 datasets click here.

This web interface allows the user to use all of the functionalities of the application, and it describes deeply the software analysis, design and implementation.

Operations

In the operations page you will find all of the operations that we implemented on the dataset

Only the operations whose execution is allowed on the active dataset are shown. The active dataset is shown at the beginning of the page, the starting dataset is the whole human genome and, through filtering operations, the dataset can be restricted.
The GFF3 dataset is never shown on its whole on the browser, in this way long waiting times and crashes are avoided.
Just above the GFF3 table some action buttons are present:

  • Dimensions of the active dataset
  • Download button: this button allows to download a .csv raw file of the dataset
  • Open in a new tab button: this button allows to show the whole GFF3 dataset in a browser tab (note that this could cause huge loading times or crashes of the browser session)

The active operations are shown in a list, each operation allows two actions:

  • Execution: the result of the operation is shown to the user in another view. if the operation is a filter, the active operations view is re-proposed, with a restricted dataset to work on. The 'operation-result' view will just display the table with the outcome of the execution and some action buttons at the top (back button, dimensions of the table and download button)
  • Documentation: this button redirects to the specific part of the documentation in which the behaviour of the operation is described in depth.
